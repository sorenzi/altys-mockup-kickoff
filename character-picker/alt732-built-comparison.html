Two picker styles, side by side

Captured directly from the iOS build. Same view (CharacterSelectionStepView), same state machine, same lore content — visual style switched via the new CharacterPickerStyle enum. Pick whichever feels right; we ship one and keep the other available for A/B or future flag.

Note: hero portraits aren't bundled yet — the gradient stage / cover-flow card show through where the character will appear. iPad landscape was forced via a temporary flag (couldn't rotate the simulator from the agent without accessibility perms) so the proportions are tall-narrow rather than wide-short — the structural intent is correct.

Pick your fighter

Cover Flow leans:
  • Maximum discoverability — kid sees both characters within ~half a second of the screen rendering
  • No chip-vs-CTA confusion — peek replaces the chip entirely
  • Familiar pattern (Apple Music, Photos)
  • Subtle background tint keeps focus on the cards
Hero Static leans:
  • More dramatic per-character moment — full-bleed gradient feels like meeting one character at a time
  • Giant name typography behind the hero is a strong visual signature
  • Story bars give explicit progress feedback for auto-advance
  • Risk: the Meet-Next chip can be missed by fast-tappers
